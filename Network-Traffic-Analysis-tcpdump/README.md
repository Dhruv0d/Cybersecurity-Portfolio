# Network Traffic Capture and Analysis using tcpdump

## Project Overview

Performed command-line network traffic capture and analysis using tcpdump on a Linux system. Captured live network traffic, filtered HTTP communications, preserved packet data in PCAP format, and conducted offline packet analysis to investigate network activity.

## Objective

* Identify network interfaces for packet capture
* Capture live network traffic from Linux systems
* Filter and monitor HTTP communications
* Save packet captures for forensic analysis
* Perform offline investigation of captured traffic

## Key Activities

### Network Interface Identification

* Enumerated available network interfaces using Linux networking utilities.
* Identified active interfaces for packet capture operations.

### Live Packet Capture

* Captured live traffic from the primary network interface.
* Inspected packet metadata including source and destination addresses, ports, TTL values, and TCP flags.

### Traffic Filtering

* Applied protocol and port-based filters to isolate HTTP traffic.
* Generated test traffic and captured relevant network communications.

### PCAP Analysis

* Saved captured packets to PCAP files for evidence preservation.
* Performed offline analysis of network traffic using tcpdump read mode.
* Examined packet payloads in hexadecimal and ASCII formats.
