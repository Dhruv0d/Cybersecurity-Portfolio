# Intrusion Detection and Alert Analysis using Suricata

## Project Overview

Configured and analyzed Suricata Intrusion Detection System (IDS) rules to detect HTTP-based network activity and investigate generated security alerts. Performed rule analysis, alert validation, and log examination using Suricata's fast.log and eve.json outputs.

## Objective

* Understand Suricata IDS rule structure
* Trigger custom detection rules against packet captures
* Analyze generated alerts
* Investigate network traffic metadata
* Examine JSON-based security event logs

## Tools Used

* Suricata IDS
* Linux Command Line
* JSON Log Analysis
* jq
* PCAP Files

## Skills Demonstrated

### Security Monitoring

* Intrusion Detection
* Alert Triage
* Security Event Analysis
* Signature-Based Detection

### Technical Skills

* Suricata Rule Analysis
* Linux CLI
* JSON Parsing
* Log Investigation
* Network Traffic Analysis

## Key Activities

### Rule Analysis

* Examined custom Suricata IDS rules.
* Analyzed rule components including actions, protocols, flow direction, signatures, and detection logic.
* Investigated how HTTP traffic is detected using content matching.

### Alert Generation

* Executed Suricata against packet capture files.
* Triggered custom IDS rules.
* Validated detection events and alert generation.

### Log Investigation

* Analyzed fast.log alert output.
* Examined eve.json structured security events.
* Investigated source IPs, destination IPs, ports, protocols, and alert metadata.

### Event Correlation

* Used flow identifiers to correlate network events.
* Extracted relevant fields from JSON logs.
* Investigated alert details for security monitoring purposes.

## Outcome

Successfully demonstrated the ability to interpret IDS rules, generate detections, investigate alerts, and analyze structured security logs. Developed practical skills directly applicable to SOC Analyst, Threat Detection, and Security Monitoring roles.
